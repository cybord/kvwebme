Templates

Product templates define how the product will look on the front-end of the site.

If you don't define a template, then all the product fields will be shown in a tabular format.

There are two templates for each type - a "multi-view" template, and a "single-view" template. The multi-view template is shown on pages where more than one product may be visible, and the single-view template is used on pages dedicated to displaying a single product.

A product template is designed much the same as a web-page template.

Let's say you have the following fields in your product type: email, name, address, phone

You can display this as a table using code like this (click the "Source" button and enter this):

<table
	<tr><th>Name</th><td>{{$name|escape}}</td></tr>
	<tr><th>Email</th><td><a href="mailto:{{$email|escape}}">{{$email|escape}}</a></td></tr>
	<tr><th>Phone</th><td>{{$phone|escape}}</td></tr>
	<tr><th>Address</th><td>{{$address}}</td></tr>
</table>

The variables names are equal, including case, to the product field names.

Notice the |escape bits. If someone enters something like great <3 stuff in the name, email or phone fields, then WebME should convert the < to &lt; before pasting to the HTML. This is called "escaping".

We don't use |escape on the address or any other textarea field, as these are usually filled in with rich-text editors so are fine to output directly.

Here's an example of the above, where the code is used in the multi-view template and shown on the front-end with two products filled in:

Normally you would have a full product defined in the single-view template, and a shortened version in the multi-view. Here's an example multi-view template:

<table style="width: 300px;">
	<tr>
		<td style="width: 100px;">{{$name|escape}}</td>
		<td style="width: 100px;">{{$phone|escape}}</td>
		<td><a href="{{PRODUCTS_LINK}}">full details</a></td>
	</tr>
</table>

In this case, we set the size of the table and table headers, as the displayed tables may differ in size otherwise.

The {{PRODUCTS_LINK}} function will generate a link to a single-view version of the product.

To display a header row above the list, edit the Header tab of your products page, and create the table header. It should use the same dimensions as the multi-view table:

<table style="width: 300px;">
	<tr>
		<th style="width: 100px;">Name</th>
		<th style="width: 100px;">Phone</th>
		<th> </th>
	</tr>
</table>

That will now render into a nicer multi-view like this:

Clicking the "full details" link will open up a products page with a single-vew version of the product.

If you don't have a page specifically for that product, then it will be rendered in the same page as was used to do the multi-view. This can cause a problem if you used a header row:

The answer is to only show the header row when we are in multi-view mode. To do this, we can use some Smarty code within the page's Header tab:

{{if $GLOBALS.products_template_used=="multiview"}}
<table style="width: 300px;">
	<tr>
		<th style="width: 100px;">Name</th>
		<th style="width: 100px;">Phone</th>
		<th> </th>
	</tr>
</table>
{{/if}}

That code will only show the header row if the template used was the multiview one.
